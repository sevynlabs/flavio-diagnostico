## Directory Purposes

**src/:**
- Purpose: All application source code and styling
- Contains: React components (.jsx), CSS stylesheets, static assets
- Key files: `SistemaCaptacao.jsx` (99.9% of application logic)

**src/assets/:**
- Purpose: Static images and SVG assets
- Contains: react.svg and other image files
- Key files: None critical (mostly placeholder assets)

**public/:**
- Purpose: Static assets served directly without bundling
- Contains: favicon, vite.svg, and other public files
- Key files: None critical (generated by Vite)

## Key File Locations

**Entry Points:**
- `index.html`: HTML document containing div#root and module script reference
- `src/main.jsx`: React DOM initialization using createRoot()
- `src/App.jsx`: Top-level React component wrapper
- `src/SistemaCaptacao.jsx`: Main application component with all page logic

**Configuration:**
- `vite.config.js`: Build tool configuration (specifies React plugin)
- `eslint.config.js`: Code quality rules (React plugins, hooks validation)
- `package.json`: Dependency management and build scripts

**Core Logic:**
- `src/SistemaCaptacao.jsx`: Entire application - all sections, state management, components, styling

**Styling:**
- `src/index.css`: Base reset and body styling (background, font smoothing)
- `src/App.css`: Legacy styles (unused or minimal usage in current design)
- Inline styles: 99% of styling via React style prop in SistemaCaptacao.jsx

## Naming Conventions

**Files:**
- PascalCase for React components: `SistemaCaptacao.jsx`, `App.jsx`
- camelCase for entry points: `main.jsx`, `vite.config.js`, `eslint.config.js`
- kebab-case for configuration: `.prettierrc` (not present but pattern expected)
- Lowercase for HTML: `index.html`

**Directories:**
- lowercase for standard directories: `src/`, `public/`
- lowercase plural for collections: `assets/`

**Functions/Variables (within components):**
- PascalCase for component names: `Icon`, `QualificationQuiz`, `SistemaCaptacao`
- camelCase for hook names: `useCounter`
- camelCase for functions: `handleAnswer`, `setShowQuiz`
- UPPER_SNAKE_CASE for constants: `WHATSAPP_LINK`

**CSS/Styling:**
- Inline style object keys use camelCase: `backgroundColor`, `fontSize`, `marginBottom`
- Color values as hex or rgb strings: `#D4AF37`, `rgba(255,255,255,0.04)`

## Where to Add New Code

**New Feature (new section/component):**
- Primary code: `src/SistemaCaptacao.jsx` - add new React component or section within main component
- Tests: Not applicable (no test infrastructure)
- Styling: Inline style object in component JSX

**New Component/Module:**
- Implementation: Create new `.jsx` file in `src/` directory (e.g., `src/SectionName.jsx`)
- Import: Import in `src/SistemaCaptacao.jsx` and use in JSX
- Styling: Inline styles or add to `src/App.css`

**Utilities/Hooks:**
- Shared helpers: Add custom hook as function in `src/SistemaCaptacao.jsx` or extract to `src/hooks.js` (new file)
- Icon additions: Add to icons object in Icon component (lines 7-26 of SistemaCaptacao.jsx)

**Static Data:**
- Arrays (testimonials, pain points, system steps): Defined as component variables in `src/SistemaCaptacao.jsx`
- Quiz questions: Defined within QualificationQuiz component (lines 66-103)
- Theme colors: Used as hex/rgb strings inline; consider extracting to `src/constants.js` for reusability

## Common Patterns for Adding Features

**Adding a new section to homepage:**
1. Create new component function in SistemaCaptacao.jsx with section-level JSX
2. Add section styling inline within the component
3. Use existing Icon component for icons or add new SVG to Icon's icons object
4. Place new section in logical flow within main JSX return statement (e.g., after TESTIMONIALS, before FINAL CTA)

**Adding interactive elements:**
1. Use useState hook at parent (SistemaCaptacao) level for state
2. Pass state and setState to child component via props
3. Use inline onClick handlers or custom event handlers
4. Style interactive states via onMouseEnter/onMouseLeave event handlers

**Styling patterns:**
1. Base structure as style object: `{ padding: "20px", borderRadius: 10, ... }`
2. Conditional styles: `background: quizDone ? "#green" : "#red"`
3. Hover states: onMouseEnter/onMouseLeave with e.target.style mutations
4. Responsive: Use CSS calc() or clamp() in style values (e.g., `fontSize: "clamp(24px, 4vw, 36px)"`)
